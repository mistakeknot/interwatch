## Architecture

interwatch is a doc freshness monitoring system for Claude Code. It abstracts the pattern: "has something changed in the project that makes this document outdated?"

### Core Abstractions

- **Watchable** ŌĆö a document that can be monitored (roadmap, PRD, AGENTS.md, etc.)
- **Signal** ŌĆö an event that might cause drift (bead closed, file renamed, version bumped)
- **Generator** ŌĆö the plugin that regenerates the doc (interpath or interdoc)
- **Confidence Tier** ŌĆö how certain we are that drift occurred (Certain/High/Medium/Low)

### Skill: doc-watch

The single skill orchestrates drift detection:

```
SKILL.md (orchestrator)
  ŌåÆ phases/detect.md   (signal evaluation)
  ŌåÆ phases/assess.md   (confidence scoring)
  ŌåÆ phases/refresh.md  (generator dispatch)
```

### Configuration

- `config/watchables.yaml` ŌĆö declarative registry of watched documents, plus `signal_templates` and `discovery_rules` for auto-discovery
- `hooks/lib-watch.sh` ŌĆö bash utilities for signal detection (git, beads, file checks)

### Auto-Discovery

The scanner auto-discovers watchable docs by convention when invoked with `--discover`:

1. Loads `signal_templates` and `discovery_rules` from the plugin's `config/watchables.yaml`
2. Resolves `{module}` placeholders from `plugin.json` name or dirname
3. Matches patterns against the project filesystem
4. Applies dedup (e.g., skips `docs/roadmap.md` if `docs/{module}-roadmap.md` exists)
5. Writes `.interwatch/watchables.yaml` with `discovered: true` entries
6. Manual entries in existing config are preserved on `--rediscover`

CLI flags: `--discover` (first-run discovery + scan), `--rediscover` (force re-detection), `--discover-only` (write config without scanning).

### State

Per-project state in `.interwatch/` (gitignored):
- `watchables.yaml` ŌĆö auto-discovered watchable config (generated by `--discover`)
- `drift.json` ŌĆö current drift scores per watchable
- `history.json` ŌĆö refresh history (when, what, confidence)
- `last-scan.json` ŌĆö snapshot baselines for change detection

## Component Conventions

### Commands

- Each has YAML frontmatter with `name` and `description`

### Hooks

- `hooks/lib-watch.sh` ŌĆö bash library (not a hook handler), provides signal detection functions

### Git hook lifecycle (Layer 1 of doc-monitoring automation)

`scripts/install-git-hooks.sh` wires `.git/hooks/post-commit` and `.git/hooks/post-merge` to refresh `.interwatch/drift.json` after every commit and merge. Runs `interwatch-scan.py --save-state` in the background with a 5s timeout ŌĆö pure-Python, zero LLM cost, never blocks the commit. Errors land in `.interwatch/hook.log`.

Properties:
- **Opt-in.** User runs `/interwatch:install-hooks` deliberately ŌĆö no auto-install on session start.
- **Idempotent.** Re-running the installer rewrites the managed block in place; never duplicates.
- **Coexistent.** Sentinel-bracketed managed block (`# >>> interwatch managed block`) preserves any existing hook content (bd-shim, husky, lefthook, etc.). Block is **prepended** after the shebang so it still runs when an existing hook ends with `exec`.
- **Reversible.** `scripts/uninstall-git-hooks.sh` removes the managed block; deletes the hook file only if it was interwatch-only.

This is the foundation of the doc-monitoring automation epic (sylveste-wdf2). Other layers (PreToolUse drift surfacing, scheduled floor refresh, audit, external replay) build on the always-fresh state this hook produces.

### Signal Evaluators

14 signal types in `scripts/interwatch-scan.py`, dispatched via `SIGNAL_EVALUATORS`:

| Signal | What it detects |
|--------|----------------|
| `bead_closed` / `bead_created` | Snapshot-delta bead counts since last scan |
| `version_bump` | plugin.json version vs. doc header mismatch |
| `component_count_changed` | Actual skill/command/agent counts vs. doc claims |
| `file_renamed` / `file_deleted` / `file_created` | Git changes since doc mtime |
| `commits_since_update` | Commit count since doc mtime (threshold-based) |
| `brainstorm_created` | Brainstorms newer than doc |
| `companion_extracted` | Companion plugins not mentioned in doc |
| `research_completed` | Research docs newer than doc |
| `roadmap_bead_coverage` | Roadmap-bead coverage via lib-watch.sh audit |
| `unsynthesized_doc_count` | Solution docs >14 days old without `synthesized_into` |
| `skills_without_compact` | SKILL.md files >90 lines lacking SKILL-compact.md |

Threshold-based signals use the `THRESHOLD_SIGNALS` dispatch table instead of per-signal `if` checks.

### Correctness Audit

Complements signal-based freshness scanning with agent-dispatched content validation. Freshness asks "has the world changed since this doc was written?" ŌĆö correctness asks "does what this doc says match reality?"

**Architecture:**
- `scripts/interwatch-audit.py` ŌĆö gathers ground truth (plugin counts, build systems, file existence, versions, prerequisites) and generates the audit agent prompt
- `commands/audit.md` ŌĆö dispatches a sonnet-tier agent with the ground truth, presents findings, offers auto-fixes

**Drift classes checked:**
1. Stale counts (hardcoded numbers vs actual)
2. Wrong commands (build commands vs available build systems)
3. Broken links (referenced files that don't exist)
4. Cross-doc consistency (conflicting facts between docs)
5. Missing prerequisites (unlisted tools needed by install/build)
6. Deprecated references (removed tools/commands still mentioned)
7. Version/output staleness (hardcoded versions vs current)
8. Ambiguous paths (file paths without clear working directory)

**State:** Audit results saved to `.interwatch/last-audit.json` for `/interwatch:status` consumption.

## Testing

```bash
cd interverse/interwatch/tests
uv run pytest -v                      # All tests
uv run pytest structural/ -v          # Structure only
uv run pytest unit/ -v                # Unit only
```

### Test Categories

- **test_structure.py** ŌĆö plugin.json validity, directory structure, marker file, lib-watch syntax, watchables config validation (including `signal_templates` and `discovery_rules`)
- **test_skills.py** ŌĆö skill count, frontmatter, phase files, reference files
- **test_commands.py** ŌĆö command count, frontmatter, expected commands exist
- **test_discovery.py** ŌĆö module name detection, auto-discovery logic, dedup, merge preservation, config writing, signal evaluators for roadmap_bead_coverage, unsynthesized_doc_count, skills_without_compact

## Development Workflow

1. Edit skill/command/config files
2. Run structural tests: `uv run pytest tests/structural/ -v`
3. Test locally: `claude --plugin-dir /root/projects/Interverse/plugins/interwatch`
4. Bump version and publish: `scripts/bump-version.sh <version>`
